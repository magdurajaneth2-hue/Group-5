Tools and Architecture Set Up

In this activity, the project was prepared by setting up the necessary tools and organizing the system structure. A code editor was used to open the project and create a main folder where all files are stored. This folder acts as the central directory of the system. A local Git repository was also initialized to track changes, making it easier to manage updates and keep different versions of the project.

A README file was created to explain the purpose of the project and to give an overview of the folder structure. The system is organized into different folders such as components, pages, services, and database to keep the project clean and easy to understand.

The system follows a simple layered architecture. The presentation layer handles the user interface, the logic layer processes the system operations, and the data layer manages the database. This structure helps make the system more organized, scalable, and easy to maintain.
